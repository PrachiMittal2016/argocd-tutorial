[#install_argocd]
== ArgoCD Installation

[tabs, subs="attributes+,+macros"]
====
Minikube::
+
--
Enable the Ingress Addon for Minikube:

[source,bash,subs="attributes+,+macros"]
----
#!/bin/bash
minikube addons enable ingress -p {profile}
minikube addons list -p {profile} | grep ingress
----

Install Argocd and check that every pod is running properly in the argocd namespace:

[source,bash,subs="attributes+,+macros"]
----
#!/bin/bash
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
kubectl get pod -n argocd
----

Patch the ArgoCD service from ClusterIP to a LoadBalancer:

[source,bash,subs="attributes+,+macros"]
----
#!/bin/bash
kubectl patch svc argocd-server -n argocd -p '{"spec": {"type": "LoadBalancer"}}'
----

ArgoCD by default generates a admin user, and a random password that generates when the ArgoCD is
deployed. To get the ArgoCD password execute:

[source,bash,subs="attributes+,+macros"]
----
#!/bin/bash
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
----

Expose the ArgoCD console using the minikube service using user admin, and the password obtained in
the step before:

[source,bash,subs="attributes+,+macros"]
----
#!/bin/bash
minikube -p {profile} service argocd-server -n argocd
----

--
OpenShift::
+
--
ArgoCD is supported in OpenShift by **Red Hat OpenShift GitOps** operator.

To install it, you need to follow the instructions written in: https://docs.openshift.com/container-platform/4.7/cicd/gitops/getting-started-with-openshift-gitops.html

It is important that after you finish with the installation, all the required pods are deployed in `openshift-gitops` namespace:

[.console-input]
[source,bash,subs="attributes+,+macros"]
----
oc get pods -n openshift-gitops
----
